API 42 Intra — Integração

Todos os endpoints da API 42 Intra que o sistema vai consumir. Base URL: https://api.intra.42.fr/v2 — Autenticação via OAuth2.

GET
/oauth/token
Troca do código de autorização por access_token. Fluxo de login OAuth2 via NextAuth.js.
access_token token_type expires_in
GET
/me
Dados completos do utilizador autenticado. Chamado no login e para sincronizar o perfil.
id login email image level campus
GET
/me/cursus_users
Progresso no cursus 42. Inclui nível, órbita actual no Holy Graph, skills adquiridas.
level grade cursus_id skills
GET
/me/projects_users
Todos os projectos do cadete. Usado para verificar atrasos e situação académica (Art. 3f).
project status final_mark validated
GET
/me/scale_teams
Histórico de avaliações. Verificar média de avaliações acima de 3 por mês (Art. 3d).
final_mark comment created_at feedback
GET
/me/coalitions
Coligação (equipa/casa) do cadete na 42 Luanda. Usado nos rankings filtrados por coligação.
id name color image_url score
GET
/me/achievements
Conquistas académicas da 42 Intra. Integradas no perfil do cadete juntamente com as conquistas do PS42.
name description image nbr_of_success
GET
/campus/:id/users
Lista de todos os cadetes do campus 42 Luanda. Usado pelo staff para sincronizar a base de cadetes.
id login level pool_year
GET
/campus/:id/exams
Exames agendados no campus. O sistema bloqueia automaticamente a PS durante exames (Art. 4e).
name begin_at end_at location
GET
/campus/:id/events
Eventos obrigatórios do campus. Bloqueia PS durante eventos de participação obrigatória (Art. 4e).
name kind begin_at max_people
GET
/users/:login/locations
Logs de localização do cadete (horas no campus). Verifica assiduidade e presença física.
host begin_at end_at
GET
/users/:id/patroned
Relações de mentoria. Identifica cadetes que são mentores — contexto extra para o perfil e rankings.
godfather godson ongoing
NOTAS DE INTEGRAÇÃO
→ Rate Limiting: A API 42 Intra tem limites de rate. Vamos fazer cache Redis de 15 minutos para dados de elegibilidade e 1 hora para dados de perfil estático.
→ Sincronização: Cron job a cada hora sincroniza dados dos cadetes. Sincronização imediata no login.
→ Campus ID: 42 Luanda tem um campus ID específico que precisamos confirmar com o staff da escola.
→ Ritmo (Art. 3c): O "ritmo ≤ 22" refere-se ao rank na fila de espera / posição no cursus — precisamos confirmar o cálculo exacto com o staff.

Motor de Elegibilidade

O sistema verifica automaticamente 8 critérios definidos no Art. 3.º do regulamento. Cada critério é verificado via API 42 Intra ou via base de dados local.

CRITÉRIO A · OBRIGATÓRIO
Ser Cadete da 42 Luanda
O utilizador deve estar inscrito no campus 42 Luanda e ter conta activa na Intra.
API: /me → campus[].id === LUANDA_CAMPUS_ID
CRITÉRIO B · OBRIGATÓRIO
2ª Órbita do Holy Graph
Ter concluído pelo menos a 2ª órbita do Holy Graph no cursus principal da 42.
API: /me/cursus_users → projects validados ≥ 2ª órbita
CRITÉRIO C · MÉTRICA
Ritmo ≤ 22
Ter um ritmo de progresso no cursus igual ou inferior ao nível 22. (Confirmar métrica exacta com staff.)
API: /me/cursus_users → level ≤ 22 OR custom metric
CRITÉRIO D · MÉTRICA MENSAL
Média de Avaliações > 3/mês
Ter uma média de avaliações (notas dadas a colegas) acima de 3 por mês nos últimos 30 dias.
API: /me/scale_teams → avg(final_mark) > 3 last 30d
CRITÉRIO E · OBRIGATÓRIO
Sem Histórico Disciplinar Grave
Não ter nenhum registo disciplinar grave activo ou histórico de infracções graves.
DB: disciplinary_records WHERE is_active=true AND type IN (suspension, expulsion)
CRITÉRIO F · CONDICIONAL
Situação Académica Regular
Sem atrasos graves nos projectos. Definido pela equipa pedagógica caso a caso.
API: /me/projects_users → sem projectos em falha crítica
CRITÉRIO G · COMPORTAMENTAL
Comportamento Adequado
Demonstrar respeito pelas regras. Avaliado com base no histórico de denúncias e incidentes.
DB: reports WHERE reported_user_id AND status = resolved (guilty)
CRITÉRIO H · OBRIGATÓRIO
Sem Sanção Disciplinar em Vigor
Não ter nenhuma sanção disciplinar activa (suspensão, admoestação com bloqueio).
DB: disciplinary_records WHERE is_active=true AND valid_until > NOW()
BLOQUEIOS AUTOMÁTICOS POR HORÁRIO E EVENTOS
HORÁRIO DE FUNCIONAMENTO
Segunda a Sexta: 08h00 às 17h00
Fora deste horário → consola bloqueada automaticamente
BLOQUEIO POR EVENTOS (API 42)
Exames → Bloqueio total
Eventos obrigatórios → Bloqueio total
Sincronizado com calendário da API 42 Intra
Stack Tecnológica

Cada tecnologia foi escolhida por uma razão específica. Nenhuma escolha arbitrária.

Framework
Next.js 14
App Router, Server Components, API Routes. Tudo num só projecto.
Linguagem
TypeScript
Tipagem forte. Erros detectados em desenvolvimento, não produção.
Styling
Tailwind CSS
Utility-first. Rápido, consistente, sem CSS global bagunçado.
Componentes
shadcn/ui
Componentes acessíveis, personalizáveis, sem dependência de biblioteca.
Autenticação
NextAuth.js
Provider OAuth2 nativo para 42 Intra. Sessões seguras, zero config.
Base de Dados
PostgreSQL
Robusto, relacional, suporta jsonb para dados flexíveis.
Hosting DB
Supabase
PostgreSQL gerido. Free tier generoso. Real-time subscriptions nativas.
ORM
Prisma
Schema como código. Migrações automáticas. Type-safe queries.
Real-time
Pusher
WebSockets geridos. Fila em tempo real, notificações push instantâneas.
Cache
Upstash Redis
Cache de elegibilidade, rate limiting, sessões temporárias. Serverless.
Email
Resend
Email transaccional moderno. API simples, templates React.
Upload
Cloudinary
Upload de fotos de incidentes. Transformações automáticas.
Gráficos
Recharts
Gráficos compostos e responsivos para o painel analytics.
Deploy
Vercel
Integração nativa Next.js. Preview deploys automáticos. Zero config.
Testes
Vitest + Playwright
Unit tests (Vitest) + E2E tests (Playwright). Cobertura completa.
